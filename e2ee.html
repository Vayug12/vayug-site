Private Subscriber-Only Video with E2EE

Vayug gives creators a way to publish exclusive subscriber-only videos using an end-to-end encrypted delivery workflow.

1

Creator Encrypts Protected Content

The creator/client side prepares the protected video and its symmetric decryption key before subscriber access is granted.

2

Keys Are Prepared Per Subscriber

The creator app can fetch public keys for active subscribers and create encrypted key material for each authorized subscriber.

3

Backend Stores Encrypted Key Material

The backend stores encrypted per-subscriber keys but does not store the raw video decryption key. An authenticated subscriber must have a matching key record.

4

Subscriber Decrypts Locally

The subscriber app obtains its authorized encrypted key and uses the E2EE playback path to decrypt the protected video locally.

Important Security & Availability Note

E2EE is intended for private or exclusive creator content. It does not make public videos private. Effective security depends on authentication, device security, private-key protection, implementation version, and current feature configuration. Creators should follow applicable privacy, copyright, and subscriber-consent requirements.
